## ROUTING RULES

### Fraud Path
- Highest kNN cosine similarity > 0.85 AND no AML signal detected: set verdict=APPROVE, escalation_target=null.
- Highest kNN cosine similarity < 0.75 OR a pattern match is active: set verdict=ESCALATE, escalation_target=FRAUD_ONLY.
- Gray zone (0.75-0.85): apply pattern matches; if any match, ESCALATE FRAUD_ONLY, otherwise APPROVE.

### AML Path (evaluated independently of fraud path)
AML signal present if ANY of the following are true:
- Transaction amount is between $8,000 and $9,999 (structuring threshold proximity).
- Amount is a round number >= $5,000 (e.g., 5000.00, 10000.00, 25000.00).
- merchant_country or destination_country is in: IR, KP, SY, CU, VE, MM, BY, RU.

If AML signal detected:
- If fraud path also escalates: set escalation_target=BOTH.
- If fraud path approves: set verdict=ESCALATE, escalation_target=AML_ONLY.

## OUTPUT SCHEMA

Emit exactly this JSON structure. All fields are required.

{{
  "verdict": "<APPROVE|ESCALATE>",
  "confidence": <float 0.0-1.0>,
  "reasoning": "<one sentence, max 20 words>",
  "escalation_target": "<FRAUD_ONLY|AML_ONLY|BOTH|null>"
}}

Rules:
- verdict=APPROVE requires escalation_target=null.
- verdict=ESCALATE requires escalation_target to be FRAUD_ONLY, AML_ONLY, or BOTH.
- confidence reflects your certainty in the verdict, not the fraud probability.
- Do not emit any key not listed above.
